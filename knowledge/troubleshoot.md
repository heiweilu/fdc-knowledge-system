# 故障分析与调试经验

## P/Q波形振荡问题

### 现象描述
仿真运行后，有功功率P或无功功率Q在稳态时仍有持续振荡，未能平滑收敛到设定值。

### 原因分析

**1. PI控制器参数不当（最常见）**

外环或内环PI参数过大会导致系统过阻尼或欠阻尼振荡。

诊断方法：
- 观察振荡频率：
  - 100Hz振荡 → 桥臂环流（2倍基频）
  - 50Hz振荡 → 内环PI过大
  - 10-30Hz振荡 → 外环与内环带宽过近
  - 5-10Hz → PLL与控制器交互振荡
  - <5Hz → 系统级功率振荡

解决方案：
```
内环PI调参步骤：
1. 设 Ki=0，逐步增大 Kp 直到阶跃响应有10%过冲
2. 记录此时临界Kp值，设 Kp = 0.6×临界Kp
3. 逐步增大 Ki 直到稳态误差消除
4. 若出现高频振荡，减小 Ki

外环PI调参步骤：
1. 确保外环带宽 ≤ 内环带宽的1/5
2. 定Udc外环：Kp=5-20, Ki=100-500
3. 定P外环：Kp=0.5-2, Ki=20-100
```

**2. 环流估抑制不足**

MMC桥臂间存在环流，主要为2倍基频分量（100Hz），会叠加在P/Q波形上。

解决方案：
- 在控制系统中加入环流抑制控制器（CCSC）
- 提取环流的2ω分量
- 通过PIR（比例积分谐振）控制器产生补偿电压

**3. 子模块电容值偏小**

电容值太小导致电压纹波大，进而引起功率波动。

诊断：查看子模块电容电压波形，纹波率>10%说明电容偏小。

解决方案：
$$C_{sm} \geq \frac{2 S_{rated}}{N \cdot \omega \cdot U_{c0}^2 \cdot \epsilon}$$

其中 $\epsilon$ 通常取5-10%。

## 直流电压不稳定

### 现象描述
MMC1站（定Udc站）的直流母线电压Udc无法稳定在设定值，出现持续波动或漂移。

### 原因分析

**1. 功率不平衡**
- 注入直流侧的功率 ≠ 从直流侧抽取的功率
- 直流母线电压反映功率平衡：$P_{in} - P_{out} = C \frac{dU_{dc}}{dt} U_{dc}$

解决方案：
- 检查MMC2的P_ref设定值是否超过系统输送能力
- 确保交流侧电源能提供足够功率
- 检查直流线路损耗是否过大

**2. 外环PI参数不匹配**
- Kp过小 → 响应慢，电压偏差大
- Kp过大 → 电压超调，可能振荡
- Ki过小 → 稳态误差不为零
- Ki过大 → 积分饱和，响应迟钝

**3. 直流侧电容不足**
- 直流母线滤波电容太小，电压纹波过大
- 建议母线电容 ≥ 5mF

**4. 子模块数量不足**
- 子模块太少导致阶梯波台阶粗，谐波注入直流侧
- 逐步增加子模块数观察效果

## 仿真发散与不收敛

### 现象描述
仿真运行后MATLAB报错"Simulation is diverging"，或数值爆炸到NaN/Inf。

### 排查流程

```
Step 1: 检查步长
  ↓ 步长 > 100μs?
  → 减小到50μs或20μs
  
Step 2: 检查代数环
  ↓ 模型诊断提示Algebraic Loop?
  → 在反馈回路中加Unit Delay
  → 或设置"配置参数→代数环→警告"为"error"来定位

Step 3: 检查初始条件
  ↓ 积分器初始值是否合理?
  → 电容初始电压设为 Udc/N
  → 电感初始电流设为0
  → 直流母线初始电压设为Udc

Step 4: 检查开环增益
  ↓ PI控制器Kp是否过大?
  → 减小到原来的50%重新运行
  
Step 5: 检查电路拓扑
  ↓ 是否存在理想元件直接并联?
  → 两个电压源不能直接并联
  → 两个电流源不能直接串联
  → 理想电压源和电容不能并联（加串联电阻）
```

### 电压源替代光伏时的特殊问题

当用直流电压源替代光伏基地时，常见问题：

1. **理想电压源与MMC直流电容冲突**
   - 原因：理想电压源的输出阻抗为0，与MMC内部电容形成环路
   - 解决：在电压源和母线间串联小电阻（0.01-1Ω）

2. **电压源与定Udc控制冲突**
   - 原因：电压源已固定Udc，MMC1又在调节Udc，两者"争抢"
   - 解决方案A：电压源设为略高于Udc_ref的值，让MMC1自然调节
   - 解决方案B：改用受控电流源模拟光伏（光伏更接近电流源特性）
   - 解决方案C：将电压源接在MMC2侧的交流端，而非直流侧

3. **初始状态不匹配**
   - 原因：仿真开始时电压源瞬间施加满压，而MMC还未建立稳态
   - 解决：电压源带斜坡启动（0→额定值，2-5s过渡）

## 启动暂态过大

### 现象描述
仿真前0.5-2s出现极大的电压/电流尖峰，远超额定值。

### 原因与解决

**1. 缺少预充电过程**

实际工程中MMC启动前需通过不控整流给子模块电容充电。

Simulink实现：
```
1. 添加一个Step信号(t=0→0, t=1s→1)
2. 将Step信号连到换流器的enable端
3. 前1s换流器闭锁，交流侧通过二极管自然充电
4. 1s后解锁换流器进入正常控制
```

具体操作：
- 在NLM调制模块的输出端加一个Switch或Product模块
- 与Step信号相乘：0-1s输出全零PWM，1s后输出正常PWM

**2. 参考值阶跃变化**

P_ref或Udc_ref从0突变到额定值会引起大的暂态。

解决：加Rate Limiter（速率限制器）
- Udc_ref：斜率限制60kV/s → 300kV需5s达到
- P_ref：斜率限制100MW/s → 500MW需5s达到

**3. 无功给定初值不为零**

如果Q_ref初始不为零，会引起额外的暂态冲击。

建议：启动阶段Q_ref=0，稳态后再调节无功。

## 仿真速度过慢

### 优化策略

**1. 减少子模块数**
- 研究初期用10-20个子模块验证控制策略
- 确认无误后再增加到工程级数量

**2. 增大仿真步长**
- 从100μs逐步增大到200μs、500μs
- 观察结果精度是否可接受
- 注意：步长太大会导致开关动作失真

**3. 使用平均值模型**
- 将MMC替换为平均值模型（Average Value Model, AVM）
- AVM忽略子模块开关细节，仿真速度提升10-100倍
- 适用于研究系统级行为（不适合研究子模块级现象）

**4. 使用加速模式**
```matlab
% 使用Simulink加速模式
set_param('MMCSIDUAN0x11', 'SimulationMode', 'accelerator')
% 或快速加速模式
set_param('MMCSIDUAN0x11', 'SimulationMode', 'rapid-accelerator')
```

**5. 并行计算**
```matlab
% 启用多核并行（需Parallel Computing Toolbox）
set_param('MMCSIDUAN0x11', 'EnableMultiTasking', 'on')
```

| 现象 | 可能原因 | 首先检查 | 调参方向 |
|------|---------|---------|---------|
| P振荡不收敛 | 外环PI过大 | Kp_outer | 减小50% |
| Q持续偏移 | Q环Ki太小 | Ki_Q | 增大2倍 |
| Udc纹波大 | 电容值偏小 | Csm | 增大到原来2倍 |
| Udc缓慢漂移 | Udc环Ki太小 | Ki_Udc | 增大 |
| 100Hz波动 | 环流未抑制 | CCSC是否启用 | 加环流抑制 |
| 高频毛刺 | 步长过大 | dt | 减小到50μs |
| 启动冲击大 | 无预充电 | 启动逻辑 | 加预充电延时 |
| 功率不匹配 | 参考值设置错误 | P_ref/Udc_ref | 检查数值 |
| 仿真中断 | 代数环/初值 | MATLAB提示 | 加延迟/设初值 |
